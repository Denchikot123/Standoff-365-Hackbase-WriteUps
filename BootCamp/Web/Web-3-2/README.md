## 🔍 Стадия 1: Разведка

После удачной эксплуатации уязвимости **SSRF** в [прошлом отчете](../Web-3-1), теперь стоит задача выполнить на узле **RCE (Remote Code Execution)**.

---

<details>
  <summary>❓Что такое RCE❓</summary>
  
>**RCE (Remote Code Execution)** - одна из самых опасный уязвимостей на данный момент, при которой злоумышленний способен удаленно на сервере выполнить вредоносный код или команды, без физического доступа.

>**Примеры атак:**

>**1. Внедрение кода через веб-формы**
>
> Хакер находит поле ввода на сайте, которое плохо проверяет данные, и вместо обычного текста (например, имени) отправляет туда кусок вредоносного серверного кода. Сервер ошибочно выполняет этот код и дает взломщику доступ к системе.
>
>**2. Эксплуатация уязвимостей в софте (Log4j / EternalBlue)**
>
> Злоумышленники используют скрытые ошибки в популярных программах или библиотеках. Например, отправляя специальный текстовый запрос серверу, они заставляют его внутренние компоненты загрузить и запустить вирус из интернета.
>
>*Для подробностей обратитесь к данному источнику [Cloudflare: What is remote code execution?](https://www.cloudflare.com/learning/security/what-is-remote-code-execution/)*

</details>

---

### Есть несколько вариантов применения применения **SSRF to RCE:**

**1. Использование нестандартных сетевых протоколов**

Если сервер принимает URL, но плохо его фильтрует, вместо http:// ему можно предоставить другие схемы:

 - ```file:///``` - для чтения локальных конфигурационных файлов.
 - ```gopher://``` or ```dict://``` - для доставки вредоносных объектов в память и автоматического выполнения кода.
 - ```ftp://``` or ```ftps://``` or ```tftp://``` - могут использоваться для обхода ограничений файрвола, сканирования внутренних портов или отправки команд на внутренние уязвимые FTP-серверы.

**2. Запрос на внутренние управляющие эндпоинты**

Вэб обращается к указанному URL от имени локального сервера (localhost). Пример запросов:

 - ```http://127.0.0.1:port/admin/run?cmd=...``` (имеет вариации для обхода [**WAF**](https://habr.com/ru/companies/selectel/articles/814183/)) - выполнение команды **Bash** на сервере *(cat /etc/passwd, whoami, id...)*

## ⚡ Стадия 2: Эксплуатация

Нашим выбором был **1 вариант**. Воспользуемся утилитой **Gopherus** для генерации вредоносной ссылки на протоколе ```gopher://```. Исходя из [прошлого отчета](../Web-3-1) было видно, что основной файл для конвертации является **convert.php**, что подтверждает использования **PHP**.

---

<details>
  <summary>❓Что такое Gopherus❓</summary>

>**Gopherus** - консольная утилита, написанная на **Python**, которая генерирует вредоносную ссылку специально для автоматизации атак класса **SSRF to RCE** через старый протокол **Gopher**.
>
>Когда вы находите на сайте уязвимость **SSRF** (сервер готов перейти по любой вашей ссылке), вам нужно заставить его не просто скачать картинку, а выполнить команду внутри своей сети. **Gopherus** генерирует уже готовую ссылку для выполнения **RCE**.

>**Пример команды:**
>gopherus --exploit {service}

>**На выбор представленно несколько сервисов:**

> - **fastcgi** - для атаки на PHP-FPM (порт 9000). Запрашивает путь к .php файлу.
> - **redis** — для атаки на базы данных Redis (порт 6379). Позволяет записать веб-шелл или задачу в cron.
> - **mysql** — для отправки запросов в незащищенную БД MySQL (порт 3306).
> - **memcached** — для атаки на хранилище Memcached (порт 11211).
> - **postgresql** — для отправки команд в базу PostgreSQL (порт 5432).
> - **zabbix** — для эксплуатации уязвимостей в агентских портах Zabbix (порт 10050).
> 
</details>

---

Для запуска, **Gopherus** потребует корректный сервис.

В нашем случае в качестве сервиса будет выступать **FastCGI**.

<img width="889" height="367" alt="изображение" src="https://github.com/user-attachments/assets/01f7f81e-eac8-452a-a227-ca8dbe764696" />

#### **Gopherus** потребует ввести некоторые данные:

- ```Give one file name which should be surely present in the server``` - Путь к абсолютно любому .php файлу. В нашем случае это ```/var/www/html/convert.php``` *(**/var/www/html/** - корень вэба)*.
- ```Terminal command to run``` - Системную консольную команду, которую нужно выполнить. В нашем случае будет выступать путь к бинарному файлу из условия - ```/home/rceflag```. *(в **Bash** это означает запуск файла)*.

В итоге утилита успешно отработала и подготовила нам вредоносную ссылку на протоколе **Gopher**.

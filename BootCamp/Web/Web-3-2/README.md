## 🔍 Стадия 1: Разведка

После удачной эксплуатации уязвимости **SSRF** в [прошлом отчете](../Web-3-1), теперь стоит задача выполнить на узле **RCE (Remote Code Execution)**

<details>
  <summary>❓Что такое RCE❓</summary>
  
>**RCE (Remote Code Execution)** - одна из самых опасный уязвимостей на данный момент, при которой злоумышленний способен удаленно на сервере выполнить вредоносный код или команды, без физического доступа.

>**Примеры атак:**

>**1. Внедрение кода через веб-формы** - Хакер находит поле ввода на сайте, которое плохо проверяет данные, и вместо обычного текста (например, имени) отправляет туда кусок вредоносного серверного кода. Сервер ошибочно выполняет этот код и дает взломщику доступ к системе.
>
>**2. Эксплуатация уязвимостей в софте (Log4j / EternalBlue)** - Злоумышленники используют скрытые ошибки в популярных программах или библиотеках. Например, отправляя специальный текстовый запрос серверу, они заставляют его внутренние компоненты загрузить и запустить вирус из интернета
>
>*Для детального изучения данной уя*

</details>
